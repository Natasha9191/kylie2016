Dernière mise à jour

Savez-vous combien de temps et d'efforts vous pouvez économiser si quelqu'un d'autre s'occupe des aspects techniques de votre site WordPress ? SiteGround est l'un des nombreux services d'hébergement géré pour WordPress qui se fera un plaisir d'effectuer ces tâches pour vous.

Mais pourquoi choisir SiteGround plutôt qu'un autre fournisseur ? Dans cet article détaillé, nous allons voir comment SiteGround se distingue de ses rivaux – ce qu'il fait mieux et ce qu'il fait moins bien – et ce que nous pensons de la solution d'hébergement WordPress proposée par ce fournisseur.

Qu'est-ce que SiteGround?

SiteGround est une entreprise d'hébergement web basée à Sofia, le hub bulgare des start-ups, qui a été fondée en 2004 par quelques amis d'université. Ce qui a commencé comme un petit projet s'est transformé en un empire mondial de l'hébergement, puisque SiteGround gère aujourd'hui des centres de données dans sept pays à travers le monde et s'occupe de l'hébergement web de 2,8 millions de domaines.

Outre les solutions d'hébergement web habituelles, SiteGround propose également un hébergement WordPress solide, que nous avons examiné à la loupe dans cet article.

Configuration & utilisation

Vous envisagez de souscrire un tarif SiteGround et vous vous demandez combien de temps prend l'installation de WordPress ? Avec SiteGround, l'installation de WordPress est vraiment simple et ne prend que quelques minutes.

Tout d'abord, vous avez besoin d'un compte. Comme un domaine est inclus dans chaque tarif SiteGround, vous pouvez l'enregistrer immédiatement après l'inscription. Vous pouvez également choisir l'emplacement de votre centre de données. Nous avons opté pour Paris.

Une fois connecté, cliquez sur « Configurer un site web » dans le portail utilisateur de SiteGround, sélectionnez WordPress dans la liste des applications dans la fenêtre suivante et suivez les instructions. SiteGround se charge ensuite de configurer votre projet WordPress, ce qui ne devrait prendre que quelques minutes.

Chez SiteGround, la configuration de WordPress est très simple.

Migrer un site vers SiteGround

Vous ne souhaitez pas créer un nouveau site web, mais migrer un site web existant vers SiteGround ? Vous avez alors deux possibilités : vous pouvez utiliser un plug-in gratuit ou contacter l'équipe de SiteGround pour qu'elle effectue la migration manuellement pour vous, moyennant des frais supplémentaires.

La migration par plug-in est assez simple : lancez d'abord la configuration du site web dans le tableau de bord SiteGround et cliquez à l'étape suivante sur « Migrer le site web ». Vous recevrez un jeton de migration dont vous aurez besoin pour la migration.

Vous pouvez migrer sans problème un site WordPress d'un autre fournisseur vers SiteGround.

Il ne vous reste plus qu'à installer le plug-in SiteGround Migrator dans le tableau de bord WordPress du site que vous souhaitez migrer. Ouvrez ensuite le plug-in, copiez le jeton de migration que vous avez reçu de SiteGround dans le champ correspondant et suivez les instructions. Le plug-in fait le reste.

Vous devez installer le plug-in SiteGround Migrator sur le site WordPress à migrer.

Fini le cPanel : Place à la nouvelle interface utilisateur de SiteGround

Voilà pour l'installation et la migration. Mais quelle est la qualité de l'interface utilisateur de SiteGround ? Commençons par l'espace client supérieur : dès que vous vous êtes connecté, vous arrivez sur une page d'accueil où quelques tutoriels vous attendent. Dans le menu principal épinglé en haut, naviguez vers l'entrée « Sites web » pour trouver l'installation WordPress que vous venez de créer. Ici, vous pouvez accéder directement à la zone d'administration de WordPress ou ouvrir les « Site Tools » (outils du site), le tableau de bord WordPress propre à SiteGround.

Jusqu'à il y a quelques années, le service utilisait le populaire tableau de bord cPanel, qui est fonctionnel (bien que pas très moderne). Entre-temps, SiteGround a développé sa propre interface utilisateur. Comme cPanel était très populaire auprès des adeptes, ce changement a suscité beaucoup d'étonnement parmi les clients. Nous pensons toutefois que le propre tableau de bord de SiteGround est bien réussi et qu'il rend la gestion de WordPress encore un peu plus facile qu'avec cPanel, surtout pour les débutants.

SiteGround utilisait auparavant le cPanel, mais a entre-temps développé sa propre interface utilisateur, spécialement adaptée à WordPress.

Site Tools : Tout en un coup d'œil

Les Site Tools (outils du site) sont également faciles à comprendre. Le tableau de bord, la page d'accueil de cette section, contient quelques outils épinglés, par exemple des liens vers l'installation de WordPress, vos comptes de messagerie et le gestionnaire de fichiers. En dessous, vous trouverez quelques statistiques.

Les liens vous permettent de naviguer vers les autres fonctions, de la gestion de vos bases de données à la mise en cache et aux sauvegardes (vous en saurez plus sur les différentes fonctions dans le chapitre suivant). Dans l'ensemble, nous n'avons pas eu à chercher longtemps un réglage.

Toutes les fonctions de WordPress sont à portée de clic dans les Site Tools.

SiteGround marque des points avec une inscription et une configuration rapides de WordPress ainsi qu'un tableau de bord convivial qui fait de l'utilisation du fournisseur un jeu d'enfant. Seul bémol : la version française n'est pas de grande qualité, ce qui laisse supposer que la traduction a été générée par une machine automatique.

Caractéristiques techniques & fonctionnalités

Si vous gérez un site web, un tableau de bord convivial est évidemment indispensable, mais pour les visiteurs de votre site, ce qui se cache sous l'interface est bien plus important. Les spécifications techniques d'un fournisseur d'hébergement géré déterminent les performances et la sécurité de vos sites WordPress.

Pour avoir une première impression de la technique utilisée, nous allons d'abord regarder le rapport sur le serveur de WordPress. Il montre les paramètres par défaut directement après l'installation. Vous le trouverez dans la zone d'administration de WordPress sous Outils > Statut du site > Rapports.

Le rapport sur le serveur directement après l'installation de WordPress chez SiteGround.

Notre installation standard de WordPress avec l'hébergeur SiteGround utilisait des serveurs web Apache, la dernière version de PHP 8.2 et une limite de mémoire PHP de 768 Mo. Les informations du rapport sur le serveur ne sont toutefois qu'un premier indicateur de la performance technique d'un service d'hébergement, car elles dépendent en fin de compte d'aspects tels que le tarif choisi et les paramètres des Site Tools.

Nous avons donc examiné de plus près les principaux détails techniques et fonctions :

Une combinaison approuvée : Nginx + Apache + MySQL

Tous les serveurs de SiteGround sont basés sur Linux (Fedora 31) et utilisent à la fois Apache et Nginx. Les bases de données proviennent de MySQL, comme pour la plupart des hébergeurs WordPress, bien que les bases de données PostgreSQL soient également prises en charge pour les charges de travail complexes. Vous pouvez créer un nombre illimité de bases de données, mais chacune est limitée à 1000 Mo.

Vous gérez vos bases de données dans les Site Tools de votre site WordPress sous Site > MySQL (ou PostgreSQL). Ici, vous pouvez créer de nouvelles bases de données et de nouveaux utilisateurs en quelques clics de souris et configurer l'accès à distance. Un accès direct à vos bases de données est possible via la console phpMyAdmin intégrée.

SiteGround supporte aussi bien MySQL que PostgreSQL.

PHP jusqu'à la version 8.2 : Nombreuses possibilités de réglage, mises à jour actuelles

La version de PHP utilisée a un impact direct sur la vitesse de chargement et la sécurité de votre site WordPress, car de nouvelles fonctionnalités sont ajoutées à chaque nouvelle version. Cependant, au moment du test, la dernière version 8.2 n'était prise en charge que par quelques hébergeurs web, dont SiteGround.

Vous pouvez accéder aux paramètres PHP sous Devs > PHP Manager. Vous avez ici la possibilité de modifier manuellement votre version PHP. Vous pouvez également activer le service Managed PHP, SiteGround actualise alors toujours automatiquement votre version PHP pour vous.

Dans les variables PHP, vous pouvez éditer les paramètres PHP en détail, bien que tous les paramètres (par exemple la limite de mémoire) ne puissent pas être adaptés manuellement.

Le mélange de services gérés adaptés aux débutants et de paramètres détaillés optionnels pour les créateurs et les développeurs est exactement ce que nous attendons d'un hébergement géré WordPress. C'est là que SiteGround marque des points.

La configuration PHP chez SiteGround.

Performance accrue grâce au SuperCacher + CDN

SiteGround optimise les performances de votre site web, notamment le temps de chargement dans les navigateurs de vos visiteurs, principalement grâce à deux fonctions : le service de mise en cache interne et un réseau de diffusion de contenu (CDN).

La mise en cache permet de stocker temporairement les éléments de votre site web et d'éviter qu'ils ne soient entièrement rechargés à chaque fois que les mêmes visiteurs accèdent au site. Le service de mise en cache de SiteGround, appelé SuperCacher, comporte trois niveaux :

1. Livraison directe via Nginx : les ressources statiques (comme les images, JS ou CSS) sont alors directement mises à disposition via Nginx, ce qui permet d'accélérer les temps de chargement.

2. Cache dynamique : il s'agit un mécanisme de mise en cache pleine page des ressources non statiques qui ne sont pas couvertes par la livraison directe.

3. Memcached : cette option de mise en cache des résultats des requêtes de la base de données améliore les temps de chargement des contenus dynamiques qui ne sont pas servis par le cache dynamique (comme les pages de passage en caisse ou les backends).

Les deux premiers niveaux sont automatiquement activés pour toutes les pages, tandis que vous devez configurer vous-même le niveau 3 pour votre application.

SiteGuard affirme que grâce à SuperCacher, un site web peut traiter jusqu'à 100 fois plus de visites qu'un site qui n'utilise pas de solution de mise en cache.

Le CDN intégré de SiteGround est pratique si votre site web attire de nombreux visiteurs internationaux. Le contenu de votre site web est stocké temporairement dans plusieurs centres de données à travers le monde. Lorsqu'un visiteur accède à votre site, le contenu est mis à disposition par un serveur aussi proche que possible, ce qui réduit les temps de chargement.

Le CDN de SiteGround améliore l'expérience utilisateur des visiteurs distants grâce à des temps de chargement rapides.

Pour savoir si SiteGround peut vraiment se démarquer de la concurrence avec des vitesses de chargement plus courtes grâce à ces fonctions, consultez le chapitre sur les performances.

Autres fonctions de SiteGround

La boîte à outils Wordpress de SiteGround est bien remplie et nous ne pouvons pas présenter chaque fonction en détail. En voici tout de même quelques-unes importantes :

  • Outil de staging : à partir du tarif GrowBig, vous pouvez créer une copie de votre site en un seul clic, ce qui vous permet de mettre en place un environnement de staging sur lequel vous pouvez tester les modifications avant de les transférer sur votre site en direct.
  • Accès SSH et SFTP : SiteGround prend en charge l'accès sécurisé aux fichiers via SSH et SFTP dans tous les tarifs.
  • WP-LCI : il existe une interface utilisateur WP-LCI nouvellement installée qui vous aide à gérer vos installations WordPress par ligne de commande.
  • Modification du fichier wp-config.php : vous pouvez modifier le fichier wp-config.php aussi bien via le gestionnaire de fichiers des Site Tools que via FTP.
  • Domaines inclus : chaque tarif comprend un domaine gratuit la première année.
  • SSD : tant pour la mémoire que pour les bases de données, SiteGround mise exclusivement sur des disques durs SSD rapides.
  • E-mails : un service de messagerie gratuit est intégré dans tous les tarifs, que vous pouvez gérer directement dans le tableau de bord de SiteGround dans une interface webmail conviviale.
  • Un PHP plus rapide : à partir du tarif GrowBig, SiteGround prend en charge sa propre solution PHP, qui devrait assurer des temps de chargement jusqu'à 30 % plus rapides.
  • Outils de collaboration : vous pouvez ajouter des collaborateurs à vos projets avec des rôles et des droits d'accès individuels.

En ce qui concerne l'installation de plug-ins, SiteGround ne vous impose aucune limite. Alors que chez de nombreux fournisseurs d'hébergement, certains plug-ins sont bloqués pour des raisons de sécurité ou de compatibilité, SiteGround vous laisse installer ce que vous voulez.

Dans l'ensemble, SiteGround a intégré de nombreuses fonctions utiles qui facilitent la vie des utilisateurs de WordPress. Les domaines et l'hébergement de messagerie intégrés font en outre de la plateforme un guichet unique pour les projets WordPress de toutes sortes.

Sécurité

Les fonctions de sécurité intégrées sont un grand avantage des solutions d'hébergement géré de WordPress : elles garantissent que votre site WordPress répond automatiquement à des normes de sécurité élevées sans que vous ayez à vous soucier vous-même de leur respect.

SiteGround promet en outre une sécurité maximale pour un confort maximal. L'isolation des comptes y veille par exemple : grâce à une configuration de serveur définie par l'utilisateur, les comptes SiteGround sont isolés les uns des autres. Les problèmes de sécurité et les points faibles d'un compte ne se répercutent donc pas sur les autres comptes.

En outre, des certificats SSL de Let's Encrypt sont intégrés gratuitement dans chaque tarif. Des certificats standard et des certificats Wildcard sont disponibles, qui sécurisent également les sous-domaines.

Chaque tarif SiteGround comprend des certificats SSL.

Pare-feu et outils de sécurité intégrés

SiteGround protège votre site web de manière proactive contre les pirates et les logiciels malveillants à l'aide d'un pare-feu d'application web (WAF) personnalisé. Il est complété chaque année par des centaines de nouvelles règles afin de garantir qu'il couvre également les vulnérabilités les plus récentes. Il y a également une protection contre le spam pour vos comptes de messagerie.

Les versions obsolètes de WordPress représentent un risque supplémentaire pour la sécurité. En tant que client de SiteGround, vous n'avez toutefois pas besoin de vous occuper vous-même de l'utilisation de la dernière version de WordPress sur votre site web, car la fonction de mise à jour automatique se charge automatiquement de la mise à jour. Si vous le souhaitez, vous pouvez également faire mettre à jour automatiquement vos plug-ins.

En outre, SiteGround propose dans les Site Tools les fonctions suivantes dans le domaine de la sécurité :

  • Forcer l'utilisation de HTTPS : en activant cette fonction, vous vous assurez que votre site web ne peut être consulté que de manière cryptée via HTTPS (tant que SSL est activé).
  • URL protégées : vous avez la possibilité de restreindre l'accès à des URL sélectionnées de votre site web à certains visiteurs à l'aide d'un mot de passe.
  • Trafic de données bloqué : vous remarquez une activité suspecte sur votre site web en provenance de certaines adresses IP ou de certains pays ? Vous pouvez alors bloquer le trafic provenant de ces adresses IP ou de ces pays afin d'éviter les attaques.
  • Analyseur de site : l'analyseur est un système payant d'alerte précoce pour les logiciels malveillants, disponible en option, qui analyse automatiquement votre site web et permet des analyses à la demande.

SiteGround répond aux principales normes de sécurité et dispose de quelques fonctions dédiées.

Sauvegardes WordPress chez SiteGround

Avec SiteGround, vous ne devez pas non plus vous occuper vous-même des sauvegardes : une sauvegarde est automatiquement créée chaque jour, ce qui vous permet de restaurer une ancienne version de votre site en cas d'urgence. Les sauvegardes sont conservées pendant 30 jours.

Les sauvegardes manuelles sont également possibles, mais pas avec le tarif le plus avantageux. Si vous souhaitez sauvegarder vous-même votre site web, vous devez choisir soit le tarif GrowBig, soit le tarif GoGeek.

Vous trouverez vos sauvegardes dans les Site Tools sous la rubrique « Sécurité ». Là, vous pouvez restaurer d'anciennes versions en quelques clics. Pour ce faire, vous pouvez soit sélectionner tous les fichiers et bases de données, soit restaurer uniquement certains fichiers, bases de données ou e-mails.

Chaque jour, une sauvegarde de votre site web est automatiquement créée et conservée pendant 30 jours.

SiteGround n'a pas encore connu d'incidents de sécurité notables. Dans l'ensemble, la plateforme remplit donc toutes les normes de sécurité que nous attendons d'un fournisseur d'hébergement géré pour WordPress et marque des points auprès des clients français avec des serveurs à Paris. À cela s'ajoutent des fonctions de sécurité pratiques et des sauvegardes automatiques avec une bonne durée de conservation de 30 jours. Il est toutefois un peu dommage que les sauvegardes manuelles ne soient pas disponibles dans tous les tarifs.

Performance

Nous avons déjà présenté les nombreuses fonctions de performance de SiteGround, il ne reste plus qu'à savoir si elles rendent votre site web plus rapide dans la pratique. C'est ce que nous avons vérifié dans notre test de performance. Pour cela, nous avons mesuré sur une période de six semaines certains des critères de performance les plus importants dans le domaine de l'hébergement WordPress.

Comparabilité des fournisseurs

Afin de pouvoir comparer au mieux les résultats des différents fournisseurs dans notre test, nous avons choisi pour chacun d'entre eux un tarif aussi proche que possible d'un prix mensuel fixé à 20 €. Chez SiteGround, le choix s'est donc porté sur le tarif StartUp, qui coûtait 18 € par mois au moment du test.

Vitesse de réponse (Time to First Byte)

Nous avons commencé par le Time to First Byte, c'est-à-dire le « temps jusqu'au premier octet ». Il mesure la vitesse de réponse du serveur en fonction de l'intervalle de temps entre l'envoi d'une requête et la réception du premier octet de données.

La vitesse de réponse peut varier pour différentes raisons, par exemple sous l'influence d'autres sites web hébergés sur le même serveur. C'est pourquoi nous l'avons testée régulièrement pendant six semaines afin d'obtenir une valeur moyenne et de noter d'éventuelles variations. Le résultat est le suivant :

Time to First Byte chez SiteGround : Bien qu'il y ait eu de nombreuses fluctuations pendant la période de test, la vitesse était en général relativement rapide.

Malgré quelques fluctuations vers le haut, SiteGround a fourni une vitesse de réponse rapide et constante par rapport à d'autres fournisseurs d'hébergement :

Vitesse de page (Largest Contentful Paint)

Nous avons ensuite mesuré le Largest Contentful Paint (LCP). Il indique le temps nécessaire à une page pour charger l'élément de contenu le plus grand ou le plus important.

Les images, CSS et Javascript sont également pris en compte, raison pour laquelle le LCP permet de tirer de bonnes conclusions sur la performance globale d'un site web. Le LCP est également important pour le SEO, car il fait partie des Core Web Vitals de Google (signaux web essentiels), grâce auxquels le moteur de recherche mesure la convivialité d'un site web.

Si vous avez un problème pour lequel vous ne trouvez pas de solution rapide, le centre d'aide SiteGround est votre premier point de contact. Vous pouvez y accéder dans le tableau de bord SiteGround en cliquant sur le point d'interrogation dans le coin supérieur droit.

Les articles sont bien classés et une fonction de recherche vous aide à trouver rapidement ce que vous cherchez. Toutefois, en lisant les articles en français, on se rend compte qu'ils n'ont pas été rédigés par de vraies personnes, mais qu'il s'agit en réalité de traductions automatiques. Les formulations des phrases sont en effet souvent étranges.

Il faut donc parfois deviner où l'article veut en venir. Le mieux est de lire directement l'article original en anglais. Mis à part les faiblesses linguistiques, le contenu du centre d'aide est plutôt bon. Il est toutefois ennuyeux qu'il réagisse un peu lentement par rapport au site régulier de SiteGround et qu'il y ait souvent des temps de chargement lorsqu'on passe d'une section ou d'un article à l'autre.

Le centre d'aide de SiteGround.

L'assistance directe est disponible via deux canaux : le chat en direct ou le ticket d'assistance. Dans le chat en direct, nous avons toujours été mis en relation presque immédiatement avec des collaborateurs. Nous avons reçu une réponse à notre ticket après environ six heures.

Bien que les collaborateurs aient répondu à nos questions en français, il était clair, une fois de plus, que les réponses étaient des textes traduits par des machines. La qualité des réponses était toutefois correcte.

Les utilisateurs francophones pourraient se sentir un peu négligés par SiteGround. Il est également dommage qu'il n'y ait pas de hotline d'assistance. En principe, le contenu du centre d'aide et les possibilités de contact sont toutefois solides.

Questions fréquentes & réponses :

Quelle est la performance de SiteGround ?
⊖

Les performances de SiteGround ont été excellentes lors du test réalisé par EXPERTE.com : dans les catégories vitesse de page et vitesse de réponse ainsi que dans le test de charge, SiteGround a devancé la plupart des autres fournisseurs que nous avons testés. La disponibilité était également bonne. Dans l'ensemble, SiteGround a fourni une meilleure performance que la plupart des fournisseurs d'hébergement.

SiteGround est-il adapté aux débutants ?
⊖

SiteGround est également adapté aux débutants. L'interface utilisateur et la gestion de WordPress sont intuitives. De plus, tout ce dont on a besoin est inclus dans les prix de SiteGround, y compris les domaines.

Puis-je exploiter plusieurs sites web avec SiteGround ?
⊖

Oui, avec SiteGround, il est possible, selon le tarif, d'exploiter plusieurs sites web. À partir de GrowBig, vous pouvez exploiter un nombre illimité de sites web. Des projets avec plusieurs sites web sont également disponibles.

SiteGround est-il sûr ?
⊖

SiteGround offre des normes de sécurité élevées, y compris SSL, pare-feu, protection contre les logiciels malveillants, sauvegardes de données et mises à jour automatiques. Jusqu'à présent, aucun incident de sécurité notable n'a été enregistré.
